I am building two focused spaces for the work I care about.

## mv-core

**Focus:** Decentralized AI compute and protocol infrastructure.

`mv-core` is where I experiment with and contribute to the stack that will run decentralized AI: consensus protocols, inference optimization, semantic caching, networking and systems programming. It includes forks and original work around AI compute, LLM serving and low-level protocol tooling.

Key areas:
- AI compute protocols (gonka-ai ecosystem)
- LLM inference (vllm, semantic caches)
- Systems programming (rustnet, bun, C-core execution)
- Protocol security and reliability

[github.com/mv-core](https://github.com/mv-core)

## mv-ml

**Focus:** Applied machine learning and intelligent systems.

`mv-ml` is the applied ML arm: document AI, OCR, computer vision, RAG, agents and the tooling that makes models useful in production. This is where prototypes turn into running systems.

Key areas:
- Document AI / OCR / RAG
- Computer vision pipelines
- Agents and MCP tooling
- ML ops and inference infrastructure

[github.com/mv-ml](https://github.com/mv-ml)
